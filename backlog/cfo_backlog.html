Document pipeline
Bug
2024 tax returns vanished after processing
M+P and Game of Thrones 2024 returns showed "processed" then disappeared. Either a display bug (stored but not surfaced) or a persistence bug (never written). High priority — if the knowledge base isn't reliably storing documents, everything built on top of it is suspect.
Bug
Chat doesn't read from extracted document data — it reasons from memory
When asked about Q1 payroll after uploading the CSV, the chat produced plausible-sounding estimates rather than reading stored facts. The pipeline needs to: (1) parse and extract structured data on upload, (2) write facts to the knowledge base, (3) make those facts available to the chat context. Right now step 3 is broken or step 2 isn't happening.
Data
Drive sweep needs a processing receipt per document
After sweep, show what was extracted: which facts, which entity, what confidence. Filing a document should confirm the knowledge base write, not just move the file. Make extraction visible, not silent.
